# Project instructions

- Use npm only and strict TypeScript. Avoid `any`, unsafe casts, and suppressed errors.
- Prefer small, reviewable modules and semantic HTML. Keyboard behavior and accessible names are required functionality; respect `prefers-reduced-motion`.
- Do not add dependencies without a clear reason. Never expose server secrets to client components or execute arbitrary user code in Next.js.
- Keep mission fixtures isolated from product UI. Keep repair, objective evaluation, and AI coaching behind interfaces.
- Never fake a passing test with timers or hard-coded success state.
- Before completion run lint, typecheck, unit tests, Playwright tests, and build.
- Do not commit generated output, secrets, test reports, browser data, node_modules, or environment files.
- Never modify files outside this repository. Commit only verified checkpoints.
